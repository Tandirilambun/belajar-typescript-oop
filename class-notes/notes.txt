Apa itu object?
- Object adalah data yang berisi field / properties / attributes dan method / function /behaviour

Apa itu Class?
- Class berupa blueprint / cetakan untuk membuat object
- Class harus ada terlebih dahulu sebelum membuat objectnya
- Class berisikan deklarasi semua properties dan function yang dimiliki oleh object 
- Setiap object selalu dibuat dari Class
- dan sebuah class bisa membuat object tanpa batas
 
OOP Typescript
- OOP tidak sedetail bahasa pemrograman lain yang memang dari awal dirancang untuk OOP seperti JAVA/C++

Constructor
- Constructor adalah methode atau function yang dipanggil ketika pertama kali object dibuat dari class
- Constructor sama saja dengan function pada umumnya, bisa memiliki parameter, bedanya constructor tidak perlu mengembalikan value

Properties atau fields
- Properties atau fields adalah attribute yang dimiliki oleh Class
- Di TS kita perlu mendeklarasikan propnya terlebih dahulu beserta tipe datanya
- Sama hanya di Type atau Interface, prop di class juga bisa dibuat optional, mandatory atau readonly
- Untuk prop yang bersifat mandatory, wajib ditambahkan nilainya di Constructornya
- untuk prop yang bersifat readonly, hanya dapat diubah didalam constructornya, diluar dari itu tidak bisa

Method
- Class juga bisa memiliki function didalamnya dan itu sering disebut sebagai method
- Cara buatnya sama seperti di JS, hanya pada TS kita harus menetukaan tipe data parameter dan return valuenya

Setter and Getter
- Saat ini bila ingin mengubah data properties bisa langsung menggunakan operator "=" dan apabila mengambil valuenya bisa menggunakan "." untuk mengambil valuenya
- Maka pada OOP ada fitur Setter dan Getter. Setter dan Getter merupakan sebauh method untuk mengubah (setter) ataupun mengambil (getter) nilai/value dari propnya
- Karena bentuknya method maka kita bisa menambahkan validasi apaun pada method tersebut sebelum mengubah nilai propnya diubah


Inheritance
- Pewarisan antara class menggunakan "extends"
- Secara otomatis semua properties dan method yang ada pada parent class akan diwariskan ke child class
- Pewarisan di TS hanya bisa memiiliki 1 parent class
- namun 1 Parent bisa memiliki banyak child class

Interface Inheritance
- Di bahasa pemrograman seperti java, kadang interface digunakan sebagai kontrak
- Interface juga dapat digunakan sebagai kontrak class dengan menggunakan kata kunci "implements"
- untuk Implements kita dapat melakukan implementasi lebih dari 1 interface, tapi pada extends hanya bisa 1 interface

Super Constructor
- Pada kasus Inheritance antar class, kadang child class ingin membuat constructor, baik itu sama seperti parent child ataupun berbeda
- Sehingga ketika membuat Constructor pada child class, maka secara otomatis kita harus memanggil constructor di parent classnya
- Menggunakan kata kunci super untuk memanggil constructor parent class

Method Overriding
- Saat membuat Child Class, kita bisa mendeklarasikan ulang method yang ada di Parent class
- apabila semua deklarasi method sama, maka itu adlah method Overriding

Super Method
- Sama halnya constructor, kita juga dapat membuat method yang sama di parent class kemudian menggunakan kata kunci super untuk menggunakan method yang ada di parent class diikuti dengan method yang ingin kita panggil

Visibility (Encapsulation)
- Pada JS dan TS, secara default dalam membuat properties atau method sifatnya bisa diakses dari dalam maupun luar class
- Dalam TS kita bisa menggunakan private, public dan protected untuk membungkus suatu properties atau method
- Public : bisa diakses darimana pun, dan secara default tidak menyebutkan visibility
- Private : hanya bisa diakses oleh classnya sendiri
- Protected : sama seperti private, tapi bisa diakses oleh class turunannya

Parameter properties
- Kadang, seringnya kita selalu membuat parameter di constructor yang hanya digunakan sebagai nilai untuk properties
- Kita bisa menggunakan Parameter Properties, yang secara otomatis parameter di Constructor akan dijadikan sebagai properties di classnya
- dalam membuat parameter properties, kita bisa langsung menambahkan visibility pada parameter di constructor


Operator Instanceof
- digunakan untuk mengecek apakah suatu object merupakan instance dari class tertentu atau bentuknya
- menggunakan typeof tidak akan bisa, karena typeof hanya mengecek tipe data
- return type dari Instanceof adalah boolean, sehingga bernilai true apabila object merupakan instance dari objectnya, dan sebaliknya

Polymorphism
- Polymorphism merupakan kemampuan sebauh object untuk berubah bentuk menjadi bentuk lain
- Polymorphism erat hubungannya dengan Inheritance

Polymorphism method
- saat membuat function/method dengan parameter, kita dapat mengirim data Polymorphism pada parameter tersebut
- Misal membuat function dengan parameter class Employee, maka kita bisa mengirim object dalam bentuk class Employee, maupun turunannya yaitu Manager dan VicePresident

Type Cast
- Kita bisa mengubah tipe data ke satu tipe data yang lain (assertion type)
- Hal ini juga bisa kita lakukan pada kasus Method Polymorphism
- Dapat dikombinasikan menggunakan instance of dan type assertion

Abstract Class
- Abstract class merupakan deklarasi Class yang belum selesai
- Sehingga diperbolehkan untuk memiliki properties atau method yang abstract atau belum dibuat code implemetasinya
- Abstract class tidak dapat dibuat menjadi object dengan menggunakan kta kunci new
- Mirip seperti Interface yang sama sama harus di turunkan (inherit)
- Bedanya dengan Interface apa?
    ~ Semua properties/method didalamnya wajib abstract
    ~ Boleh sebagian ada non abstract dan abstract
- Abstract Class bertujuan untuk dijadikan sebagai Parent Class yang akan diturunkan
- Abstract Class seperti menggambarkan hubungan antar Object Parent dan Object Child,
- sedangkan Interface lebih ke arah kemampuan atau apa yang bisa dilakukan oleh sebuah object

Static
- Static merupakan kata kunci yang bisa digunakan pada properties atau method di class
yang menyebabkan properties atau method tersebut bukan lagi bagian dari object yang dibuat dari class
- Static baik method ataupun properties, akan berperilaku seperti kita membaut global variable/function,
sehingga daapt diakses langsung tanpa membuat object dari classnya sendiri
- Tapi kita tetap bisa menambahkan visibility pada static properties/method
- Paling sering kita jumpat untuk digunakan sebagai class Utility/Helper

Class Relationship
- Karena implementasi dari object di TypeScript adalah JS Object
- Jadi sebenarnya jika terdapat dua object walaupun berbeda class, tetapi secara prop dan function tetap sama
- Sehingga kita bisa membuat object untuk tipe data A, dan membuat object dari tipe data B asal prop dan method sama

Error handling
- Sama halnya di JS, di TS pun mendukung error handling menggunakan try catch
- Cara penggunaannya sama seperti TS
- Kita juga dapat membuat class Error secara manual dengan membuat class turunan dari Error

Namespace
- Namespace digunakan untuk mengorganisir kode ketika dalam satu module terdapat banyak sekali kode sehingga dikelola dalam Namespace
- Maka jika dianggap Modeul ini sebuah folder, maka Namespace ini adalah subfolder didalamnya
- dapat dibuat dengan kata kunci namespace dan kita bisa menambahkan class, function dll didalam Namespace tersebut
